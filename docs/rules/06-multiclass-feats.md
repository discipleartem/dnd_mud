# Глава 6: Индивидуальные опции

> Источник: PHB, стр. 163–170. Полный текст — `docs/PHB_ D&D_2023 RUS.pdf` (локально).

## Для игроков

### Мультиклассирование

> **В dnd_mud (текущий этап):** мультиклассирование **запрещено**. У персонажа один класс; при повышении уровня класс не меняется.

При повышении уровня можно взять уровень **другого класса** вместо текущего. Ограничения:
- минимальные значения характеристик для нового класса;
- владения и ячейки заклинаний считаются по **особым таблицам** мультикласса;
- бонус мастерства и атаки — по суммарному уровню персонажа, умения — по уровням в каждом классе.

### Черты (feats)

На уровнях, где класс даёт **увеличение характеристики** (`ability_score_improvement` в `classes.yaml`), можно вместо этого взять **черту** — постоянное правило (например, «Атлет», «Бдительный»). Variant human на 1 уровне получает одну черту на шаге создания после предыстории.

## Для разработчиков

### Политика MUD (текущий этап)

**Мультиклассирование запрещено.** У персонажа ровно один класс (`Character.class_name`); при повышении уровня класс не меняется, уровни второго класса не накапливаются. Правила PHB в разделе «Для игроков» — справочные; реализация мультикласса не планируется до отдельного решения по roadmap.
