# Глава 6: Мультиклассирование

> Источник: PHB, стр. 163–165. Полный текст — `docs/PHB_ D&D_2023 RUS.pdf` (локально).  
> См. также: [Черты (feats)](06-feats.md) — отдельная опция PHB из той же главы.

## Для игроков

> **В dnd_mud (текущий этап):** мультиклассирование **запрещено**. У персонажа один класс; при повышении уровня класс не меняется.

При повышении уровня по правилам PHB можно взять уровень **другого класса** вместо текущего. Ограничения:

- минимальные значения характеристик для нового класса;
- владения и ячейки заклинаний считаются по **особым таблицам** мультикласса;
- бонус мастерства и атаки — по суммарному уровню персонажа, умения — по уровням в каждом классе.

## Для разработчиков

### Политика MUD (текущий этап)

**Мультиклассирование запрещено.** У персонажа ровно один класс (`Character.class_id`); при повышении уровня класс не меняется, уровни второго класса не накапливаются. Правила PHB в разделе «Для игроков» — справочные; реализация мультикласса не планируется до отдельного решения по roadmap.

| Аспект | Реализация |
|--------|------------|
| Статус | **Запрещено** (Pre-Alpha) |
| Модель | Одно поле `class_id`; нет списка классов/уровней |
| UI | Не добавлять шаг выбора второго класса в создание и левелап |
| YAML | Таблицы мультикласса в PHB — не загружаются |

### Запланировано (если появится в roadmap)

| Аспект | Целевая реализация |
|--------|-------------------|
| Хранение | Список пар `(class_id, level)` или отдельная структура прогрессии |
| Требования к характеристикам | Проверка при добавлении класса |
| Владения при мультиклассе | Особые правила PHB (частичное владение) |
| Ячейки заклинаний | Таблица мультиклассового заклинателя |
| Левелап | Выбор: текущий класс +1 или новый класс 1 |
| UI | Шаг после ASI/feat или отдельный пункт меню левелапа |

До принятия решения по roadmap мультикласс **не смешивать** с чертами в коде и flow — см. [06-feats.md](06-feats.md).
