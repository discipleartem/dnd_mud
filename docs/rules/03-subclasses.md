# Подклассы (архетипы) PHB

> Источник: PHB, глава 3 (классы), стр. 48–119. Полный текст — `docs/PHB_ D&D_2023 RUS.pdf` (локально).

Подкласс (архетип, домен, круг, клятва, происхождение, покровитель, школа магии) — специализация внутри класса. В PHB у каждого из 12 классов **два или три** варианта (у жреца — семь доменов, у волшебника — восемь школ). Выбор обычно делается на **1–3 уровне**; дальнейшие умения подкласса открываются по уровням класса.

Общие сведения о классах — [03-classes.md](03-classes.md). Создание персонажа — [01-character-creation.md](01-character-creation.md).

### Политика MUD

**В игре реализуются только подклассы, перечисленные в** [`database/classes/classes.yaml`](../../database/classes/classes.yaml) **для соответствующего класса.** Сводная таблица PHB ниже — справочник по правилам D&D 5e, а не полный план разработки. Подклассы PHB, отсутствующие в YAML, не входят в scope ядра (расширение — через моды `type: addon` после появления класса в каталоге).

## Для разработчиков

| Аспект | Значение |
|--------|----------|
| Статус | **Phase 2** — выбор подкласса не в flow создания; умения не применяются в runtime |
| Канон каталога | Только `subclasses` из [`classes.yaml`](../../database/classes/classes.yaml) для классов, уже присутствующих в YAML |
| PRD | §3.4.6: выбор подкласса из каталога, навыки класса, ячейки — запланированы |
| Целевой flow | После выбора класса или при левелапе (`level` ≥ порога подкласса); список вариантов — `load_classes()` → `subclasses` выбранного класса |

Новый подкласс в игре = запись в `classes.yaml` (и при необходимости мод). Реализация механики PHB для ID, которых нет в YAML, **не планируется** в ядре.

### Рекомендуемая схема данных (Phase 2)

```yaml
subclasses:
  - id: battle_master
    name: { ru: "Мастер боевых искусств", en: "Battle Master" }
    parent_class: fighter
    choice_level: 3
    phb_pages: "72-74"
    features:
      - id: battle_maneuvers
        level: 3
        feature_type: active
```

Поле `choice_level` должно соответствовать таблице выше. Домены жреца и происхождения чародея/колдуна в PHB выбираются на **1 уровне** класса.

### Не реализовано

- UI выбора подкласса при создании или левелапе
- Применение `features` подкласса в game engine
- Заклинания доменов, клятв, расширенные списки колдуна
- Кости превосходства, приёмы, дикий облик, зверь-спутник и прочая механика подклассов
