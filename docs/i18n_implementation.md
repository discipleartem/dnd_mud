# Реализация системы интернационализации (i18n)

## Обзор

Создана полнофункциональная система локализации для D&D Text MUD с поддержкой множества языков, кэшированием и автоматическим определением языка системы. Система включает комплексные тесты (52 теста) и полную интеграцию с UI компонентами.

## Архитектура

### Основные компоненты

1. **entities/i18n.py** - Основная сущность I18n
   - `I18nConfig` - Конфигурация системы
   - `LanguageInfo` - Информация о языке
   - `TranslationKey` - Ключ перевода с контекстом
   - `I18n` - Основной класс системы

2. **interfaces/i18n_api.py** - Интерфейсы системы
   - `I18nLoader` - Загрузчик переводов
   - `I18nDetector` - Детектор языка системы
   - `I18nTranslator` - Интерфейс переводчика
   - `I18nManager` - Менеджер локализации
   - `I18nValidator` - Валидатор переводов

3. **use_cases/i18n_manager.py** - Реализации интерфейсов
   - `YamlI18nLoader` - Загрузчик YAML файлов
   - `SystemI18nDetector` - Автоопределение языка
   - `DefaultI18nTranslator` - Базовый переводчик
   - `DefaultI18nValidator` - Валидатор по умолчанию
   - `I18nManagerImpl` - Основной менеджер

## Использование

### Базовая инициализация

```python
from pathlib import Path
from entities.i18n import I18nConfig
from use_cases.i18n_manager import I18nManagerImpl

# Создание менеджера
data_dir = Path("data")
manager = I18nManagerImpl(data_dir)

# Конфигурация
config = I18nConfig(
    default_language="ru",
    fallback_language="en",
    cache_enabled=True,
    auto_detect_language=True,
)

# Инициализация
manager.initialize(config)
manager.load_all_translations()
```

### Работа с переводами

```python
# Получение переводчика
translator = manager.get_translator()

# Простые переводы
welcome = translator.translate("welcome")

# С контекстом
title = translator.translate("title", context="ui.main_menu")

# С параметрами
msg = translator.translate("greeting", name="Мир")

# Переключение языка
translator.set_language("en")
```

### Интеграция с UI

```python
from ui.console import Console

# Создание консоли с переводчиком
console = Console(translator)

# Локализованные методы
console.print_title(console.t("ui.main_menu.title"))
console.print_info(console.t("welcome"))
```

### Локализация сущностей

```python
from entities.character import Character

hero = Character("Арагорн")
hero.set_translator(translator)

# Локализованные методы
print(hero.get_status())  # "Здоров" / "Healthy"
print(str(hero))         # "Арагорн (Новичок, HP: 45/55)"
```

## Языковые файлы

### Формат YAML

```yaml
# ru.yaml
welcome: "Добро пожаловать в D&D Text MUD!"
ui:
  main_menu:
    title: "D&D Text MUD"
    new_game: "Новая игра"
    load_game: "Загрузить игру"
    settings: "Настройки"
    exit: "Выход"

errors:
  number_expected: "Ожидалось число"
  range_error: "Значение должно быть от {min} до {max}"

# characters.yaml - вложенная структура для персонажей
character:
  status:
    dead: "Погиб"
    healthy: "Здоров"
    wounded: "Ранен"
    heavily_wounded: "Тяжело ранен"
levels:
  novice: "Новичок"
  adventurer: "Авантюрист"
```

### Поддержка контекстов

- `ui.*` - Элементы интерфейса
- `character.*` - Персонажи и их атрибуты
- `errors.*` - Сообщения об ошибках
- `combat.*` - Боевые действия
- `items.*` - Предметы и снаряжение

### Примеры тестовых сценариев

```python
def test_character_with_translator(self) -> None:
    """Тест локализации персонажа."""
    config = I18nConfig()
    i18n = I18n(config)
    translator = DefaultI18nTranslator(i18n)
    
    translations = {
        "character": {
            "status": {
                "dead": "Погиб",
                "healthy": "Здоров",
            },
        },
        "levels": {
            "novice": "Новичок",
        },
    }
    i18n.load_translations("ru", translations)
    i18n.set_language("ru")
    
    character = Character("Тестовый персонаж", hit_points=0, max_hit_points=10)
    character.set_translator(translator)
    
    assert character.get_status() == "Погиб"
```

## Расширение

### Добавление нового языка

1. Создать файл `data/i18n/{code}.yaml`
2. Добавить язык в `YamlI18nLoader.get_available_languages()`
3. Перевести все ключи из базового файла
4. Добавить тесты для нового языка

### Добавление нового контекста

1. Добавить секцию в YAML файлы
2. Использовать `context="название"` при переводе
3. Обновить документацию
4. Добавить соответствующие тесты

### Падежи и формы
