# Архитектура проекта

## Обзор архитектуры

D&D Text MUD построен на принципах чистой архитектуры с четким разделением ответственности между модулями.

## Структура директорий

## Основные компоненты

### 1. Ядро игры (Core)

**Game** — главный класс, управляющий состоянием игры:
- Инициализация систем
- Управление игровым циклом
- Координация между модулями

**Config** — управление конфигурацией:
- Загрузка настроек из YAML
- Управление параметрами игры
- Валидация конфигурации

### 2. Пользовательский интерфейс (UI)

**Console** — консольный интерфейс на основе Rich:
- Отрисовка экранов
- Обработка ввода пользователя
- Управление цветовой схемой

**Menus** — система меню:
- Главное меню
- Меню создания персонажа
- Настройки

**Screens** — игровые экраны:
- Экран персонажа
- Инвентарь
- Карта мира

### 3. Управление данными (Data)

**Loader** — загрузка YAML данных:
- Парсинг YAML файлов
- Кеширование данных
- Обработка ошибок

**Models** — Pydantic модели:
- Структуры данных для валидации
- Типизация данных
- Сериализация/десериализация

### 4. Система персонажей (Character)

**Character** — основной класс персонажа:
- Характеристики и навыки
- Инвентарь и экипировка
- Заклинания и способности

**Race** — система рас:
- Расовые черты
- Бонусы к характеристикам
- Особые способности

**Class** — система классов:
- Классовые способности
- Прогрессия уровней
- Архетипы

### 5. Боевая система (Combat)

**Combat** — управление боем:
- Порядок инициативы
- Ходы участников
- Расчет результатов

**Actions** — действия в бою:
- Атаки и защита
- Применение заклинаний
- Использование способностей

### 6. Приключения (Adventure)

**Adventure** — структура приключений:
- Сцены и локации
- События и триггеры
- Прогрессия сюжета

**Dialogue** — система диалогов:
- Варианты ответов
- Проверки навыков
- Последствия выбора

### 7. Система модов (Mods)

**Manager** — управление модами:
- Загрузка и активация
- Проверка совместимости
- Разрешение конфликтов

**Loader** — загрузка контента модов:
- YAML файлы модов
- Интеграция с основным контентом
- Валидация модов

### 8. Система сохранения (Save)

**SaveManager** — управление сохранениями:
- Создание и загрузка сохранений
- Управление слотами
- Автосохранение

**Serializer** — сериализация данных:
- Преобразование объектов в JSON
- Восстановление состояния игры
- Валидация сохранений

### 9. Локализация (i18n)

**Translator** — система перевода:
- Перевод текстовых строк
- Поддержка множественных языков
- Форматирование строк

**Loader** — загрузка локализации:
- JSON файлы переводов
- Кеширование переводов
- Обработка отсутствующих переводов

## Поток данных

### 1. Запуск игры

```
main.py → Game.__init__() → Config.load() → DataLoader.load_all()
```

### 2. Создание персонажа

```
UI → CharacterCreator → Race/Class/Background → Character.save()
```

### 3. Игровой цикл

```
Game Loop → Adventure.process() → UI.render() → User Input → Game Loop
```

### 4. Бой

```
Adventure.start_combat() → Combat.init() → Combat.turn() → Combat.resolve()
```

## Принципы проектирования

### 1. Чистая архитектура
- Бизнес-логика независима от UI
- Инверсия зависимостей
- Тестируемость компонентов

### 2. SOLID принципы
- **S**ingle Responsibility — каждый класс имеет одну ответственность
- **O**pen/Closed — открыто для расширения, закрыто для изменения
- **L**iskov Substitution — подклассы заменяют базовые классы
- **I**nterface Segregation — маленькие, специфичные интерфейсы
- **D**ependency Inversion — зависимость от абстракций

### 3. DRY и KISS
- Избегание дублирования кода
- Простота реализации
- Читаемость кода

### 4. Type Safety
- Type hints везде
- Pydantic для валидации
- MyPy для проверки типов
