# Product Requirements Document (PRD)
## Текстовая MUD-игра по мотивам Dungeons & Dragons 5-й редакции

**Версия документа:** 1.0  
**Стадия:** Pre-Alpha  
**Платформы:** Windows, Linux  
**Инструменты:** Python 3.12, colorama, pyproject.toml  
**БД:** YAML-файлы (папка `database/`)

---

## 1. Цель проекта
Создать модульную, расширяемую текстовую приключенческую игру в жанре MUD, основанную на правилах D&D 5e, с фокусом на одиночное прохождение через выбор вариантов из нумерованных списков. Игра должна легко модифицироваться пользователями без навыков программирования через YAML-файлы.

## 2. Объем (Pre-Alpha)
Реализуются:
- Приветственный экран (ASCII-art + colorama)
- Главное меню (8 пунктов)
- Окно/терминал с динамическим переносом текста (Window Manager)
- Создание персонажа (имя, раса, класс, уровень 1, базовые характеристики)
- Меню «Настройки» (размер шрифта, Hard Core режим)
- Меню «Languages» (русский/английский, библиотека локализации)
- Меню «Модификации» (просмотр и активация .yaml модов)
- Меню «Приключения» (2 встроенных: обучение, Lost Mine of Phandelver)
- Базовая логика выбора персонажа и старта приключения

## 3. Функциональные требования

### 3.1. Приветственный экран
- Выводится при запуске игры.
- Текст: «Добро пожаловать в текстовую игру по мотивам Dungeons and Dragons 5 edition»
- Ниже — версия игры (из `pyproject.toml` или отдельного конфига).
- Использовать цветные ASCII-границы через `colorama`.

### 3.2. Главное меню
Нумерованный список пунктов:

1. Новая игра  
2. Загрузить игру  
3. Создать персонажа  
4. Настройки  
5. Languages  
6. Модификации  
7. Приключения  
8. Выход

**Поведение:**
- После выбора пункта открывается соответствующий экран.
- «Новая игра»:  
  - Если нет ни одного персонажа → переход к экрану создания персонажа.  
  - Если есть один или более персонажей → показать список существующих (имя, уровень, раса, класс) для выбора.  
  - После выбора персонажа → переход в меню «Приключения».
- «Загрузить игру»: загрузка сохранённого состояния (пока заглушка, но структура должна быть).
- «Создать персонажа» → отдельный экран.
- «Выход» → завершение программы.

### 3.3. Экран создания персонажа
- Запрашивается имя (строка, не менее 2 символов, уникальное в пределах сохранённых персонажей).
- Выбор расы из ограниченного списка (хранится в `database/races.yaml`):  
  - Человек, Эльф, Дварф, Халфлинг (на Pre-Alpha).
- Выбор класса (из `database/classes.yaml`):  
  - Воин, Жрец, Плут, Волшебник.
- Характеристики генерируются стандартным массивом D&D 5e (15,14,13,12,10,8) с расовыми бонусами.
- После создания персонаж сохраняется в `database/characters.yaml`.

### 3.4. Меню «Настройки»
- **Размер шрифта** (пользовательский ввод: маленький/средний/большой).  
  По умолчанию средний. Влияет на размер текста в терминале (через соответствующие escape-последовательности, если терминал поддерживает).
- **Режим Hard Core** (True/False).  
  По умолчанию False. Влияет на правила игры (например, перманентная смерть, ограничение сохранений).

### 3.5. Меню «Languages»
- Используется библиотека локализации (например `gettext` или `babel`).  
- Язык по умолчанию: русский.  
- Доступные языки (список всегда на английском):  
  1. English  
  2. Русский  
- При смене языка интерфейс и тексты приключений (если есть локализация) переключаются.

### 3.6. Меню «Модификации»
- Сканирует папку `mods/` на наличие YAML-файлов.
- Каждый мод должен содержать поля:  
  - `name`, `version`, `type` (`addon` или `mod`), `description`, `files` (список YAML-файлов для замены/дополнения).
- `addon` — официальное расширение (добавляет новые расы, классы, приключения).  
- `mod` — изменяет существующую логику (например, переназначает правила).
- Интерфейс: отображать список модов с возможностью включить/выключить.  
- Включённые моды применяются при следующем запуске игры (или перезагрузке ядра).

### 3.7. Меню «Приключения»
- Список доступных приключений, загруженных из `database/adventures.yaml` и модов типа `addon`.
- Два встроенных:  
  1. **Обучение (Tutorial)**  
     - Сложность: лёгкая  
     - Тип: обучение  
  2. **Затеряные рудники Фанделвера (Lost Mine of Phandelver)**  
     - Сложность: нормальная  
     - Тип: приключение
- Каждое приключение имеет:  
  - `id`, `name`, `description`, `difficulty` (easy/normal/hardcore), `type`, `script` (ссылка на YAML-файл сценария).
- При выборе приключения запускается игровой цикл (ядро).

## 4. Нефункциональные требования

### 4.1. Модульность
- Ядро (core) не зависит от UI, БД, модов.
- Core предоставляет API для:
  - Получения данных о персонаже
  - Выполнения действия (бросок кубика, проверка навыка, получение описания комнаты)
  - Управления состоянием игры (сохранение/загрузка)
- UI (меню, ввод/вывод) — отдельный слой.

### 4.2. Обработка ввода
- По умолчанию все выборы — через ввод числа (пункт меню, действие).
- Исключения: ввод имени персонажа, пути к файлу мода, текст поиска (в будущем).  
- Валидация ввода с повторным запросом.

### 4.3. Window Manager (адаптивный перенос текста)
- Определение текущей ширины терминала (через `shutil.get_terminal_size()` или `os`).
- Все тексты, выводимые в UI, автоматически разбиваются на строки не длиннее `width-2` (с учётом границ).
- В настройках «размер шрифта» может влиять на расчёт отступов (но на уровне отображения – просто сохранение настройки для будущих UI-расчётов).

### 4.4. Производительность
- Загрузка YAML-файлов при старте игры (или при смене языка/модов).
- Игровой цикл не должен задерживать ввод (отсутствие `time.sleep` без причины).

### 4.5. Локализация
- Все пользовательские тексты вынесены в `.po`/`.mo` или YAML-словари.
- Ключи локализации должны быть осмысленными.
- При смене языка перезагружаются все текстовые ресурсы.

6. Архитектура (модули)
6.1. Core (ядро)
game_engine.py – главный цикл игры: получает команду от UI, обращается к модели, обновляет состояние, возвращает текст ответа.

character.py – класс Character (имя, раса, класс, уровень, хиты, статы), методы save(), load(), create().

dice.py – функции roll_d20(), roll(ndm).

adventure.py – класс Adventure, загружает YAML-сценарий, управляет текущей комнатой/узлом, проверяет условия, выдаёт варианты действий.

mod_loader.py – читает папку mods/, мержит YAML-словари в соответствии с action (append, replace, delete).

localization.py – обёртка над gettext, функция _(), переключение языка.

6.2. UI
window_manager.py – функция print_wrapped(text, width=None), автоматическое определение ширины, учёт цветов.

menus.py – класс Menu, методы display_main_menu(), display_settings(), display_languages() и т.д. Возвращают выбранный пункт.

input_handler.py – функция get_int_input(prompt, min_val, max_val) и get_str_input(prompt, validator).

6.3. Entry point (main.py)
Инициализация colorama, определение размера терминала, загрузка настроек, локализация, модов.

Запуск основного цикла меню (пока не выбран выход).

7. Пользовательский интерфейс
Все экраны центрированы или выровнены по левому краю с рамкой (ASCII-символы).

Каждый пункт меню предваряется номером.

Цвета: заголовки — жёлтый, ошибки — красный, успех — зелёный (через colorama).

После выбора действия экран перерисовывается (очистка терминала).

9. Ограничения и допущения
Одиночная игра (без сети).

Нет поддержки многопользовательской игры.

Движок боя — упрощённый (D20 + модификаторы).

Состояние игры сохраняется только вручную (автосохранение не требуется на Pre-Alpha).

Графика отсутствует (только текст).

Моды не требуют перезагрузки интерпретатора (но могут требовать перезагрузки ядра через вызов функции reload_mods).

10. Критерии приёмки Pre-Alpha
При запуске main.py отображается приветствие с версией.

Все пункты главного меню открывают соответствующие экраны (даже если внутри заглушки «в разработке»).

Можно создать персонажа: ввести имя, выбрать расу и класс. Персонаж сохраняется в YAML.

При выборе «Новая игра» отображается список созданных персонажей (если есть).

Настройки «Размер шрифта» и «Hard Core» сохраняются в YAML и считываются при запуске.

Смена языка меняет тексты интерфейса на русский/английский.

При наличии папки mods/ с корректным YAML-файлом меню «Модификации» показывает его и позволяет включить/выключить (запоминает состояние).

Весь текст корректно переносится при изменении размера терминала (в том числе динамически, если игра перерисовывает экран после изменения размера).
