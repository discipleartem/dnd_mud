# Product Requirements Document (PRD)
## Текстовая MUD-игра по мотивам Dungeons & Dragons 5-й редакции

**Версия документа:** 1.1
**Стадия:** Pre-Alpha
**Платформы:** Windows, Linux
**Инструменты:** Python 3.12, colorama, PyYAML, pyproject.toml
**БД:** YAML-файлы (папка `database/`, 18 файлов)

---

## 1. Цель проекта
Создать модульную, расширяемую текстовую приключенческую игру в жанре MUD, основанную на правилах D&D 5e, с фокусом на одиночное прохождение через выбор вариантов из нумерованных списков. Игра должна легко модифицироваться пользователями без навыков программирования через YAML-файлы.

## 2. Объем (Pre-Alpha — текущее состояние)

### Реализовано
- Приветственный экран (ASCII-art + colorama)
- Главное меню (2 пункта: Настройки, Выход)
- Экран «Настройки» (переключение Hard Core режима)
- База данных правил D&D 5e (18 YAML-файлов: расы, классы, доспехи, оружие, навыки, языки, фиты, особенности и т.д.)
- Локализация (русский/английский, YAML-словари)
- Броски кубиков (d20, ndm, преимущество/помеха, критические попадания)
- Модель персонажа (Character dataclass, создание, сохранение/загрузка)
- Загрузка приключений из YAML
- Модуль модов (сканирование, включение/выключение)
- Игровой движок (базовая структура)
- Адаптивный вывод текста с переносом по ширине терминала

### Запланировано (следующие итерации)
- Полноценное главное меню (8 пунктов: Новая игра, Загрузить, Создать персонажа, Настройки, Languages, Модификации, Приключения, Выход)
- Меню создания персонажа (имя, раса, класс, характеристики)
- Меню Languages (выбор языка интерфейса)
- Меню Модификаций (просмотр и активация .yaml модов)
- Меню Приключений (выбор сценария)
- Полноценный движок приключений (комнаты, проверки, бой)
- Система сохранения игрового состояния
- Режим Hard Core (перманентная смерть)

## 3. Функциональные требования

### 3.1. Приветственный экран
- Выводится при запуске игры.
- Текст: «Добро пожаловать в текстовую игру по мотивам Dungeons and Dragons 5 edition»
- Ниже — версия игры (из `pyproject.toml` или отдельного конфига).
- Использовать цветные ASCII-границы через `colorama`.

### 3.2. Главное меню (целевое)
Нумерованный список пунктов:

1. Новая игра
2. Загрузить игру
3. Создать персонажа
4. Настройки
5. Languages
6. Модификации
7. Приключения
8. Выход

**Текущее состояние:** Пункты 4 (Настройки) и 8 (Выход) работают. Остальные — в разработке.

### 3.3. Экран создания персонажа (в разработке)
- Запрашивается имя (строка, не менее 2 символов, уникальное в пределах сохранённых персонажей).
- Выбор расы из списка (хранится в `database/races.yaml`).
- Выбор класса (из `database/classes.yaml`).
- Характеристики генерируются стандартным массивом D&D 5e (15,14,13,12,10,8) с расовыми бонусами.
- После создания персонаж сохраняется в `database/characters.yaml`.

### 3.4. Меню «Настройки» (реализовано)
- **Режим Hard Core** (True/False). По умолчанию False. Влияет на правила игры (например, перманентная смерть, ограничение сохранений).

### 3.5. Меню «Languages» (в разработке)
- Используется самописная библиотека локализации на YAML-словарях (не gettext/babel).
- Язык по умолчанию: русский.
- Доступные языки:
  1. English
  2. Русский
- При смене языка интерфейс переключается.

### 3.6. Меню «Модификации» (в разработке)
- Сканирует папку `mods/` на наличие YAML-файлов.
- Каждый мод должен содержать поля: `name`, `version`, `type` (`addon` или `mod`), `description`, `files`.
- `addon` — официальное расширение (добавляет новые расы, классы, приключения).
- `mod` — изменяет существующую логику.
- Интерфейс: отображать список модов с возможностью включить/выключить.

### 3.7. Меню «Приключения» (в разработке)
- Список доступных приключений, загруженных из `database/adventures.yaml`.
- Два встроенных:
  1. **Обучение (Tutorial)** — сложность: лёгкая, тип: обучение
  2. **Затеряные рудники Фанделвера (Lost Mine of Phandelver)** — сложность: нормальная, тип: приключение
- При выборе приключения запускается игровой цикл (ядро).

## 4. Нефункциональные требования

### 4.1. Модульность
- Ядро (core) не зависит от UI, БД, модов.
- Core предоставляет API для:
  - Получения данных о персонаже
  - Выполнения действия (бросок кубика, проверка навыка, получение описания комнаты)
  - Управления состоянием игры (сохранение/загрузка)
- UI (меню, ввод/вывод) — отдельный слой.

### 4.2. Обработка ввода
- По умолчанию все выборы — через ввод числа (пункт меню, действие).
- Исключения: ввод имени персонажа, пути к файлу мода, текст поиска (в будущем).
- Валидация ввода с повторным запросом.

### 4.3. Производительность
- Загрузка YAML-файлов при старте игры (или при смене языка/модов).
- Игровой цикл не должен задерживать ввод (отсутствие `time.sleep` без причины).

### 4.4. Локализация
- Все пользовательские тексты вынесены в YAML-словари (папка `database/strings/`).
- Ключи локализации используют точечную нотацию для вложенных словарей.
- При смене языка перезагружаются все текстовые ресурсы.
- Fallback на английский язык, если ключ не найден.

### 5.3. Форматы YAML

**database/races.yaml**
```yaml
races:
  human:
    name:
      ru: "Человек"
      en: "Human"
    ability_bonuses:
      strength: 1
      dexterity: 1
      constitution: 1
      intelligence: 1
      wisdom: 1
      charisma: 1
```

**database/classes.yaml**
```yaml
classes:
  fighter:
    name:
      ru: "Воин"
      en: "Fighter"
    hit_die: 10
    primary_ability: strength
```

**database/characters.yaml**
```yaml
characters:
  - name: "Арагорн"
    level: 1
    race: human
    class: fighter
    stats:
      strength: 15
      dexterity: 14
      constitution: 13
      intelligence: 12
      wisdom: 10
      charisma: 8
    current_hp: 12
    experience: 0
```

**database/adventures.yaml**
```yaml
adventures:
  - id: tutorial
    name:
      ru: "Обучение"
      en: "Tutorial"
    type: training
    difficulty: easy
    script_file: "adventures_scripts/tutorial.yaml"
  - id: lost_mine
    name:
      ru: "Затеряные рудники Фанделвера"
      en: "Lost Mine of Phandelver"
    type: adventure
    difficulty: normal
    script_file: "adventures_scripts/lost_mine.yaml"
```

### 5.4. Формат модификации (mods/example_mod.yaml)
```yaml
name: "Новая раса - Драконорожденный"
version: "1.0"
type: "addon"   # addon или mod
description: "Добавляет расу Dragonborn"
files:
  - target: "database/races.yaml"
    action: "append"   # или "replace" / "delete"
    data:
      - id: dragonborn
        name:
          ru: "Драконорожденный"
          en: "Dragonborn"
        ability_bonuses:
          strength: 2
          charisma: 1
```

## 6. Архитектура (модули)

### 6.1. Core (ядро)
- `game_engine.py` – главный цикл игры: получает команду от UI, обращается к модели, обновляет состояние, возвращает текст ответа.
- `character.py` – класс Character (dataclass: имя, раса, класс, уровень, хиты, статы), методы to_dict()/from_dict(), функции create_character(), load_characters(), save_characters().
- `dice.py` – функции roll(), roll_d20(), roll_with_mods(), critical_hit().
- `adventure.py` – загрузка приключений из YAML, отображаемые имена.
- `mod_loader.py` – сканирование папки mods/, включение/выключение модов.
- `localization.py` – класс Localization (YAML-словари с fallback на английский).
- `settings.py` – класс Settings (язык, hardcore).

### 6.2. UI
- `menus.py` – функции отрисовки экранов (show_welcome_screen, show_main_menu, show_settings).
- `input_handler.py` – get_int_input(), get_str_input(), get_choice().

### 6.3. Entry point (main.py)
- Инициализация colorama, загрузка настроек, локализация.
- Запуск основного цикла меню (пока не выбран выход).

## 7. Пользовательский интерфейс

Все экраны центрированы или выровнены по левому краю с рамкой (ASCII-символы).

Каждый пункт меню предваряется номером.

Цвета: заголовки — жёлтый, ошибки — красный, успех — зелёный (через colorama).

После выбора действия экран перерисовывается (очистка терминала).

```
3. Создать персонажа
4. Настройки
5. Languages
6. Модификации
7. Приключения
8. Выход

Выберите пункт [1-8]:
```

## 9. Ограничения и допущения

- Одиночная игра (без сети).
- Нет поддержки многопользовательской игры.
- Движок боя — упрощённый (D20 + модификаторы).
- Состояние игры сохраняется только вручную (автосохранение не требуется на Pre-Alpha).
- Графика отсутствует (только текст).
- Моды не требуют перезагрузки интерпретатора (но могут требовать перезагрузки ядра через вызов функции reload_mods).
