# D&D Text MUD

Однопользовательская консольная текстовая игра по мотивам механики Dungeons & Dragons 5-й редакции.

## Текущий этап разработки

**Этап:** Приветственное окно, Главное меню, навигация (MVP)

### Что реализовано:
- ✅ Приветственный экран
- ✅ Главное меню с навигацией (Новая игра, Загрузить игра, Настройки, Выход)
- ✅ Базовое создание персонажа (имя, уровень, здоровье)
- ✅ Консольный интерфейс с цветовой поддержкой
- ✅ Чистая архитектура с разделением слоёв
- ✅ Следование принципам Zen Python (KISS, YAGNI, DRY)
- ✅ Оптимизированный код после рефакторинга
- ✅ Улучшенная читаемость и соответствие PEP 8

### Что в планах:
- ⏳ Расширенное создание персонажа (расы, классы, предыстории)
- ⏳ Приключения и диалоги
- ⏳ Боевая система
- ⏳ Система прогрессии

## Структура проекта

```
src/
├── main.py              # Точка входа
├── core/
│   ├── game.py         # Основной класс игры (координация)
│   └── constants.py    # Общие константы проекта (DRY)
├── entities/
│   └── character.py    # Сущность персонажа (бизнес-логика)
├── use_cases/
│   └── game.py         # Use Cases меню (сценарии использования)
├── ui/
│   └── console.py      # Консольный интерфейс (адаптер)
└── interfaces/
    └── user_interface.py  # Интерфейс UI (абстракция)
```

## Принципы разработки

Проект следует:
- **Zen Python**: Простое лучше сложного, читаемость важна
- **KISS**: Максимально простые решения
- **YAGNI**: Только необходимая функциональность  
- **DRY**: Никакого дублирования кода
- **Чистая архитектура**: Чёткое разделение слоёв

### Архитектурные особенности

- **Entities** не зависят от UI, базы данных или внешних сервисов
- **Use Cases** содержат бизнес-логику и зависят только от Entities
- **Interface Adapters** преобразуют данные между слоями
- **Frameworks** (консоль) находятся во внешнем слое
- **Constants** вынесены в отдельный модуль для избежания дублирования
