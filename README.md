## Обзор проекта

D&D MUD - это консольная текстовая RPG игра, реализованная на Python с применением принципов чистой архитектуры. Проект следует современным практикам разработки и использует актуальные инструменты для обеспечения качества кода.

### Основные характеристики

- **Язык**: Python 3.12+
- **Архитектура**: Чистая архитектура (Clean Architecture)
- **Интерфейс**: Консольный терминал
- **Данные**: YAML файлы для игрового контента
- **Тестирование**: pytest с покрытием кода
- **Форматирование**: black, ruff
- **Проверка типов**: mypy

## Архитектура

### Чистая архитектура

Проект построен по принципам чистой архитектуры с чётким разделением ответственности:

```
src/
├── models/entities/          # Бизнес-сущности (Character, Race, Class)
├── use_cases/              # Сценарии использования (CreateCharacter, LoadGame)
├── interfaces/             # Интерфейсы внешних зависимостей
├── adapters/               # Адаптеры интерфейсов (контроллеры)
├── repositories/           # Реализации репозиториев (файловые)
├── console/               # Консольный интерфейс
└── dtos/                  # DTO для передачи данных
```

### Правила зависимостей

- **Entities** → не зависят ни от чего
- **Use Cases** → зависят только от Entities
- **Interface Adapters** → зависят от Use Cases и Entities  
- **Repositories & Console** → зависят от всех внутренних слоев

### Поток данных

1. **Console Layer** получает ввод пользователя
2. **Adapters** преобразуют ввод в DTO
3. **Use Cases** обрабатывают бизнес-логику
4. **Repositories** работают с данными
5. **Entities** представляют бизнес-сущности

## Установка и запуск

### Требования

- Python 3.12 или выше
- pip (менеджер пакетов Python)

### Шаги установки

1. **Клонирование репозитория**
   ```bash
   git clone https://github.com/discipleartem/dnd_mud.git
   cd dnd_mud
   ```

2. **Создание виртуального окружения**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Linux/Mac
   # или
   .venv\Scripts\activate  # Windows
   ```

3. **Установка зависимостей**
   ```bash
   pip install -e .
   ```

4. **Установка зависимостей для разработки (опционально)**
   ```bash
   pip install -e ".[dev]"
   ```

### Запуск игры

```bash
python main.py
```

Или через установленную команду:

```bash
dnd-mud
```

## Разработка

### Инструменты разработки

Проект использует следующие инструменты для обеспечения качества кода:

- **black** - форматирование кода
- **ruff** - линтинг и проверка стиля
- **mypy** - статическая проверка типов
- **pytest** - написание и запуск тестов
- **pytest-cov** - измерение покрытия кода тестами

### Команды разработки

```bash
# Форматирование кода
black src/ tests/

# Линтинг
ruff check src/ tests/

# Проверка типов
mypy src/

# Запуск тестов
pytest

# Запуск тестов с покрытием
pytest --cov=src --cov-report=html
```

### Структура данных

Игровые данные хранятся в формате YAML в директории `data/`:

- `abilities.yaml` - способности персонажей
- `backgrounds.yaml` - предыстории персонажей
- `characters.yaml` - шаблоны персонажей
- `classes.yaml` - классы персонажей

## Тестирование

### Запуск тестов

```bash
# Все тесты
pytest

# С подробным выводом
pytest -v

# С покрытием кода
pytest --cov=src --cov-report=html

# Конкретный тест
pytest tests/test_specific.py::test_function
```

### Написание тестов

Тесты должны находиться в директории `tests/` и следовать структуре `src/`:

```
tests/
├── test_entities/
├── test_use_cases/
├── test_adapters/
└── test_repositories/
```

## Правила кода

### Стиль кода

- **Длина строки**: 79 символов
- **Отступы**: 4 пробела
- **Именование**: snake_case для переменных и функций, PascalCase для классов
- **Документация**: docstrings в Google Style на русском языке

### Типизация

- Все функции должны иметь type hints
- Используется mypy для проверки типов
- Запрещено использовать `py.typed`

### Принципы

- **KISS** (Keep It Simple, Stupid)
- **DRY** (Don't Repeat Yourself)
- **YAGNI** (You Aren't Gonna Need It)
- **SOLID** принципы ООП
